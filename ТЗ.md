# ТЗ: Viral Content Manager — платформа для продвижения приложений через органический контент

## Контекст

Инструмент для одного пользователя, помогающий системно продвигать мобильные приложения (Brainura и другие) через органический контент в TikTok и Instagram. Основан на методологии курса "Виральный контент" (Максим Артюшенков): парсинг конкурентов, анализ хуков и скриптов, создание AI-креаторов, генерация контента (карусели, видео), трекинг гипотез и публикаций.

**Пользователь:** один (без авторизации)
**Язык интерфейса:** русский
**AI-чат:** не нужен (используется Claude Code отдельно)
**Деплой:** Vercel

---

## Стек технологий

| Слой | Технология |
|---|---|
| Фреймворк | Next.js 15 (App Router, Server Actions) |
| UI | shadcn/ui + Tailwind CSS 4 |

| Фоновые задачи | Inngest (бесплатный тир: 5000 шагов/мес) |
| Деплой | Vercel |

## Модули и функциональность

### 1. Проекты (Projects)

Мультипроектная структура — каждое приложение = отдельный проект. Все данные (аккаунты, хуки, гипотезы) привязаны к проекту.

**Страница:** `/projects`

**Поля проекта:**
- `name` — название (например, "Brainura")
- `description` — описание продукта и ЦА
- `app_store_url` — ссылка на App Store
- `play_store_url` — ссылка на Google Play
- `website_url` — сайт/лендинг
- `product_doc` — текстовое поле (Product Doc: описание продукта, ЦА, боли, позиционирование)
- `target_platforms` — TikTok, Instagram, оба
- `target_regions` — целевые регионы (US, RU, EU и т.д.)
- `created_at`, `updated_at`

**Функции:**
- CRUD проектов
- Переключатель текущего проекта в хедере (глобальный контекст)
- Дашборд проекта — сводка по количеству аккаунтов, видео, хуков, гипотез

---

### 2. Отслеживаемые аккаунты (Tracked Accounts)

Аккаунты конкурентов и вдохновения, которые парсятся и анализируются.

**Страница:** `/accounts`

**Поля:**
- `project_id` — FK на проект
- `platform` — tiktok / instagram
- `username` — @username
- `url` — полная ссылка
- `display_name` — отображаемое имя
- `bio` — описание профиля
- `followers_count` — подписчики
- `following_count` — подписки
- `videos_count` — кол-во видео
- `avg_views` — средние просмотры
- `avg_engagement_rate` — средний ER
- `category` — категория (competitor / inspiration / niche_leader / own)
- `tags` — теги (string[], например: ["food", "ai-influencer", "carousel"])
- `notes` — заметки
- `last_scraped_at` — когда последний раз парсили
- `scrape_status` — idle / in_progress / error
- `created_at`, `updated_at`

**Функции:**
- Добавить аккаунт по URL → автоматический парсинг через ScrapeCreators
- Таблица с фильтрами: платформа, категория, теги, сортировка по метрикам
- Детальная страница аккаунта: профиль + список его видео + паттерны хуков
- Кнопка "Обновить данные" — повторный парсинг
- Массовый парсинг (по расписанию через Inngest)

---

### 3. Видео / Контент (Videos)

Спарсенные видео с метриками. Основной массив данных для анализа.

**Страница:** `/videos`

**Поля:**
- `account_id` — FK на аккаунт
- `platform` — tiktok / instagram
- `video_id` — ID на платформе
- `url` — ссылка на видео
- `type` — video / carousel / photo
- `thumbnail_url` — превью
- `description` — описание/caption
- `duration_seconds` — длительность
- `views_count` — просмотры
- `likes_count` — лайки
- `comments_count` — комментарии
- `shares_count` — репосты
- `saves_count` — сохранения
- `engagement_rate` — ER (рассчитывается)
- `posted_at` — дата публикации
- `hashtags` — хештеги (string[])
- `music_name` — название музыки
- `is_analyzed` — проанализировано AI или нет
- `analysis` — JSON с результатами AI-анализа (хук, скрипт, визуал, инсайты)
- `hook_text` — извлечённый текст хука (первые слова)
- `hook_visual` — описание визуала в момент хука
- `full_script` — полный скрипт/транскрипт видео
- `tags` — теги (string[])
- `notes` — заметки
- `is_bookmarked` — в избранном
- `created_at`

**Функции:**
- Таблица с фильтрами: аккаунт, тип, дата, метрики, теги, избранное
- Сортировка по просмотрам, ER, дате
- Детальная карточка видео: превью + метрики + хук + скрипт + AI-анализ
- Кнопка "Проанализировать" → отправляет в Inngest → Gemini API → сохраняет результат
- Массовый анализ топ-N видео аккаунта
- Добавить видео по URL (без привязки к аккаунту — для разовых находок)

### 4. Библиотека хуков (Hooks Library)

Извлечённые и каталогизированные хуки — ключевой актив для создания контента.

**Страница:** `/hooks`

**Поля:**
- `project_id` — FK на проект
- `video_id` — FK на видео (опционально, если извлечён из видео)
- `text` — текст хука
- `visual_description` — что в кадре
- `hook_type` — question / statement / shock / curiosity / callout / story / list
- `language` — язык хука
- `source_views` — просмотры исходного видео (для оценки эффективности)
- `source_er` — ER исходного видео
- `adapted_text` — адаптированный текст под мой продукт
- `tags` — теги
- `rating` — 1-5, личная оценка
- `notes` — заметки
- `is_used` — использован ли уже
- `created_at`

**Функции:**
- Таблица/карточки с фильтрами: тип, язык, рейтинг, теги, использован/нет
- Быстрое добавление хука вручную
- Автоматическое извлечение из видео при AI-анализе
- Поле "Адаптация" — ручной ввод адаптированной версии хука под свой продукт

---

### 5. Библиотека скриптов (Scripts Library)

Полные сценарии видео для референса и адаптации.

**Страница:** `/scripts`

**Поля:**
- `project_id` — FK
- `video_id` — FK (опционально)
- `title` — название/тема скрипта
- `hook` — хук (первая фраза)
- `body` — основная часть
- `cta` — призыв к действию
- `full_text` — полный текст
- `format` — talking_head / voiceover / demo / carousel_script / skit
- `duration_seconds` — примерная длительность
- `language` — язык
- `source_views` — просмотры оригинала
- `adapted_version` — адаптированная версия под мой продукт
- `tags` — теги
- `rating` — 1-5
- `notes` — заметки
- `is_used` — использован ли
- `created_at`

**Функции:**
- Таблица с фильтрами: формат, язык, рейтинг
- Автоматическое заполнение из AI-анализа видео
- Ручное добавление и редактирование

---

### 6. AI-креаторы (Creators)

Профили AI-персонажей для генерации контента (по методологии курса — Creator Doc).

**Страница:** `/creators`

**Поля:**
- `project_id` — FK
- `name` — имя креатора
- `prototype_account_id` — FK на аккаунт-прототип (опционально)
- `summary` — кто этот креатор (2-3 предложения)
- `appearance` — детальное описание внешности
- `voice_and_speech` — голос и манера речи
- `personality` — характер и подача
- `background` — жизнь, работа, хобби
- `visual_style` — окружение, фоны, одежда
- `image_gen_prompt` — промпт для генерации изображений (английский)
- `reference_images` — URL-ы референсных изображений (string[])
- `generated_images` — URL-ы сгенерированных изображений (string[])
- `top_hooks` — JSON: топ хуки прототипа
- `top_scripts` — JSON: топ скрипты прототипа
- `status` — draft / active / archived
- `notes` — заметки
- `created_at`, `updated_at`

**Функции:**
- Карточки креаторов с аватаром и описанием
- Создание креатора вручную
- Создание на основе аккаунта-прототипа (парсинг + AI-анализ через Inngest)
- Редактирование всех полей
- Экспорт Creator Doc в Markdown

**Страница:** `/trends`

**Поля:**

- `example_urls` — примеры (string[])
- `relevance` — hot / warm / cold
- `applicability` — насколько подходит моему продукту (1-5)
- `adaptation_notes` — как адаптировать
- `tags` — теги
- `created_at`, `updated_at`

**Функции:**
- Карточки/список с фильтрами: тип, платформа, актуальность
- Быстрое добавление идеи
- Связь с видео-примерами из базы

### 8. Гипотезы и тесты (Hypotheses)

Контент-гипотезы: что хотим протестировать, какой результат получили.

**Страница:** `/hypotheses`

**Поля:**
- `project_id` — FK
- `title` — короткое описание гипотезы
- `description` — детальное описание: что тестируем и почему
- `format` — формат контента (carousel / short_video / long_video / photo)
- `hook_id` — FK на хук (опционально)
- `script_id` — FK на скрипт (опционально)
- `creator_id` — FK на креатора (опционально)
- `status` — idea / in_production / published / analyzed
- `priority` — high / medium / low
- `expected_result` — что ожидаем
- `actual_result` — что получили
- `publication_url` — ссылка на опубликованный контент
- `published_at` — дата публикации
- `metrics` — JSON: просмотры, лайки, комменты, шеры, сохранения, ER
- `learnings` — выводы (что узнали)
- `tags` — теги
- `created_at`, `updated_at`

**Функции:**
- Канбан-доска: idea → in_production → published → analyzed
- Таблица с фильтрами: статус, приоритет, формат
- После публикации — ручной или автоматический ввод метрик
- Поле "Выводы" — фиксация инсайтов

## Интеграции (внешние API)

### ScrapeCreators API
- **Цель:** парсинг аккаунтов и видео TikTok/Instagram
- **Эндпоинты:**
  - Account Analysis — получить профиль и список видео с метриками
  - Video Analysis — получить детали конкретного видео
- **Где используется:** модули Accounts и Videos
- **Ключ:** хранится в переменных окружения (`SCRAPECREATORS_API_KEY`)

### Gemini API (Google AI Studio)
- **Цель:** AI-анализ видео (извлечение хуков, скриптов, визуальный анализ)
- **Модель:** Gemini 3 Flash (или Pro для глубокого анализа)
- **Где используется:** анализ видео → заполнение полей hook_text, hook_visual, full_script, analysis
- **Ключ:** `GEMINI_API_KEY`

### Runware API (опционально, фаза 2)
- **Цель:** генерация изображений для каруселей и креаторов
- **Где используется:** модуль Creators (генерация аватаров), генерация карусельных слайдов
- **Ключ:** `RUNWARE_API_KEY`

## Фоновые задачи (Inngest)

| Задача | Триггер | Описание |
|---|---|---|
| `scrape-account` | Кнопка "Добавить/Обновить аккаунт" | Парсит профиль + все видео через ScrapeCreators |
| `analyze-video` | Кнопка "Проанализировать" | Отправляет видео в Gemini, сохраняет результат |
| `batch-analyze` | Кнопка "Анализировать топ-N" | Цепочка: берёт видео по очереди → analyze-video |
| `scrape-all-accounts` | Cron (раз в неделю) | Обновляет метрики всех отслеживаемых аккаунтов |
| `create-creator-doc` | Кнопка "Создать из прототипа" | Парсит аккаунт → анализирует топ видео → формирует Creator Doc |

Каждая задача разбита на шаги (steps), каждый шаг < 60 секунд → укладывается в лимиты Vercel.

## Схема базы данных (Prisma)

```prisma
model Project {

  websiteUrl      String?
  productDoc      String?  @db.Text
  targetPlatforms String[] // ["tiktok", "instagram"]
  targetRegions   String[] // ["US", "RU"]
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  accounts   Account[]
  hooks      Hook[]
  scripts    Script[]
```
